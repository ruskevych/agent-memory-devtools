# MVP Product Language

## Positioning

Use this headline-level positioning:

> Debuggable, correctable memory system for AI coding agents.

The product should feel like a real devtool: local, inspectable, scriptable, and honest about limits.

## What To Emphasize

- Developers can see what the agent remembered and why.
- Developers can replay ingestion and retrieval decisions.
- Developers can correct bad memory through feedback and Memory Fix Mode.
- Missing Memory Analysis finds likely durable context that ingestion skipped.
- Confidence and conflict signals help developers decide what to trust.
- SQLite and deterministic local embeddings keep the default workflow local-first.

## What Not To Claim

- No hosted sync or cloud account.
- No auth, teams, enterprise policy, or remote vector database dependency.
- No full UI rule-management screen yet.
- No broad claim that every conflict type is deeply resolved; current conflict detection is conservative and focused on obvious preference contradictions.

## Terminology

Use these names consistently:

- Memory Fix Mode
- Missing Memory Analysis
- replay traces
- feedback
- rules
- missing-memory suggestions
- confidence
- conflicts
